# Assets y placeholders

## Filosofía
Los placeholders deben parecer una decisión de diseño premium temporal, no ausencia de material final.

## Lista de assets a preparar

### 1. Hero desktop
- Ruta objetivo futura: `/public/images/hero/hero-desktop.webp`
- Tamaño recomendado: 1600 x 1200 px
- Estilo: industrial cinematográfico premium
- Contenido sugerido: composición abstracta o semi-realista de fluidos, componentes, tuberías, superficies metálicas, válvulas o sistemas de proceso con iluminación dirigida

### 2. Hero mobile
- Ruta objetivo futura: `/public/images/hero/hero-mobile.webp`
- Tamaño recomendado: 1080 x 1350 px
- Debe funcionar con recorte vertical sin perder impacto

### 3. About visual
- Ruta: `/public/images/sections/about-visual.webp`
- Tamaño recomendado: 1200 x 900 px
- Estilo: ingeniería aplicada, soporte, interacción técnica o detalle de componente

### 4. Industrias placeholders
- Ruta: `/public/images/industries/*.webp`
- 6 a 8 imágenes
- Estilo consistente, oscuro, premium, técnico

### 5. Fabricantes / ecosistema
- Ruta: `/public/images/partners/*.svg|webp`
- Logos oficiales cuando exista permiso de uso
- Si no, usar logo + fondo premium y pequeñas descripciones

### 6. Casos de aplicación
- Ruta: `/public/images/use-cases/*.webp`
- Mostrar aplicaciones o superficies / activos / reparación / protección

### 7. Open Graph
- Ruta: `/public/og/og-image.jpg`
- Tamaño: 1200 x 630 px
- Fondo oscuro premium + headline + marca

## Sistema placeholder premium si aún no hay fotos finales
Crear piezas abstractas con:
- malla técnica tenue
- gradiente oscuro
- líneas fluidas
- brillos controlados
- overlays metálicos / cyan / steel
- textura mínima

## Evitar
- stock cliché de casco amarillo estrechando manos
- engranes genéricos
- fábricas borrosas de mala calidad
- imágenes saturadas o muy naranjas
